Why is my copy of Windows feature limited?

Since this version of Windows was built entirely with CSS, you may notice some features are missing. This is because of the limitations of CSS. To learn more about CSS, you can click here.

Why can't I move windows?

Although CSS lets you do quite a lot of things, unfortunately drag and drop is not one of them. To enable drag and drop on this version of windows, please learn Javascript first, and then write the necessary scripts to add drag and drop.

Why can't I move windows?

Creating Windows in CSS is quite difficult, and requires a lot of input elements. Therefore, it may be recommended that you use another language if you wish to make a fully fledged operating system. Even so, it is a good way to demonstrate the power of CSS.

How long did it take to make CSS Windows?

Roughly 1.5 weeks, on and off.

Database Enhancement

Artifact Overview

The Corner Grocer app now uses a persistent SQLite backend, replacing in-memory storage. All previous features remain, but data persists across runs for real-world utility and portability.

Justification for Inclusion

This enhancement integrates a database while maintaining backward compatibility. It demonstrates schema design, SQL queries, and Python’s DB-API, improving reliability and distribution without external setup.

Skills & Course Outcomes

Outcome 1 — Collaboration

  • Dedicated db_handler.py isolates database logic
  • Structured comments explain SQL choices
  • Clear, readable function names

Outcome 2 — Communication

  • Menu options for DB actions are clearly labeled
  • Success and error messages improve user trust
  • Documentation clarifies data flow

Outcome 3 — Algorithmic/Design Choices

  • Efficient queries for storing/retrieving frequencies
  • Separation of concerns between DB and UI
  • Function abstraction for maintainability and tests

Outcome 4 — Techniques & Tools

  • SQLite integration elevates value and reliability
  • Reusability of DB functions across menu paths
  • Backward compatibility preserved

Outcome 5 — Security Mindset

  • Validated item names reduce bad data and risk
  • try/except guards connection/query failures
  • Clear error messages without exposing internals

Reflection

Shifting to SQLite required re-thinking data flow while preserving the interface. The biggest wins were cleanly separating DB concerns and improving feedback to users. The result is a more reliable, maintainable application.
